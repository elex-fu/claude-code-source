# Claude Code 小白入门指南

## 一、Claude Code 是什么？

Claude Code 是 **Anthropic** 推出的 AI 编程助手，它直接运行在你的**终端（Terminal）**里，可以帮你：

- 💬 **聊天式编程** - 用自然语言描述需求，AI 帮你写代码
- 📁 **操作文件** - 读取、编辑、创建、搜索文件
- 🔧 **运行命令** - 执行 bash 命令、git 操作、运行测试
- 🔍 **分析代码** - 理解项目结构、查找 bug、代码审查
- 🌐 **联网搜索** - 查找文档、获取最新信息

最重要的是：**它就在你的终端里，能直接访问你的代码和开发环境**。

---

## 二、为什么要用 Claude Code？

### 对比传统方式

| 场景 | 传统方式 | Claude Code |
|------|----------|-------------|
| 写新功能 | 查文档 → 写代码 → 调试 | 描述需求，AI 直接生成 |
| 改 bug | 看日志 → 猜原因 → 试修复 | AI 分析代码，定位问题 |
| 重构代码 | 手动改几十个文件 | AI 批量修改，你确认即可 |
| 学习新项目 | 花几小时看代码 | AI 总结项目结构 |

### 对比网页版 ChatGPT/Claude

- **知道你的项目**: 能读取本地文件，不是「瞎子摸象」
- **能执行命令**: 不只是给建议，真的能改文件、跑测试
- **在终端里**: 不用切换窗口，编程体验更流畅

## 三、基本概念（不用懂代码也能看懂）

### 3.1 工具（Tools）

Claude Code 有「超能力」—— 工具。当它需要做事时，会调用相应的工具：

```
你想："帮我看看这个文件"
AI 调用：FileRead 工具 → 读取文件内容 → 告诉你内容

你想："运行测试"
AI 调用：Bash 工具 → 执行 npm test → 把结果给你

你想："搜索代码"
AI 调用：Grep 工具 → 搜索关键词 → 列出匹配位置
```

**常见工具**：
- 📖 **FileRead** - 读文件
- ✏️ **FileEdit** - 改文件（会显示改动对比）
- 📝 **FileWrite** - 写新文件
- 🔍 **Grep** - 搜索内容
- 📂 **Glob** - 查找文件（如 "所有 .js 文件"）
- 💻 **Bash** - 执行命令
- 🌐 **WebFetch** - 获取网页内容
- 🤖 **Agent** - 创建子助手（复杂任务分工）

### 3.2 权限模式

Claude Code 很注重安全，不会让你意外被坑：

| 模式 | 说明 | 适合场景 |
|------|------|----------|
| **Ask** (默认) | 每次改文件/执行命令都问你 | 初学者、重要项目 |
| **Auto** | 只读操作自动执行，写入仍询问 | 日常使用 |
| **Plan** | AI 先做计划，你确认后再执行 | 大改动 |
| **Accept** | 自动接受大部分操作 | 信任的项目 |
| **Bypass** | 几乎全自动化（谨慎！） | 临时快速操作 |

### 3.3 记忆系统

Claude Code 能记住你的偏好：

- **CLAUDE.md** - 放在项目里，告诉 AI 项目规则
- **Session Memory** - 当前会话的记忆
- **Global Memory** - 跨项目的个人偏好

例如，你可以在 CLAUDE.md 里写：
```
我们项目使用 2 空格缩进
提交前要运行 eslint
不要修改 .env 文件
```

## 四、常用操作

### 4.1 启动 Claude Code

```bash
# 在项目目录中打开
claude

# 直接打开（后续再选项目）
claude

# 快速提问后直接退出
claude -p "帮我检查这个项目的依赖版本"
```

### 4.2 基本对话

```
# 询问代码
"这个函数是做什么的？"

# 让 AI 改代码
"把这个函数改成异步的"

# 分析错误
"运行报错了，帮我看看"

# 批量操作
"把所有 console.log 改成 logger.debug"
```

### 4.3 特殊命令（以 / 开头）

| 命令 | 作用 |
|------|------|
| `/help` | 查看帮助 |
| `/cost` | 查看本次花费 |
| `/compact` | 压缩对话历史（太长时用） |
| `/clear` | 清空当前对话 |
| `/exit` 或 `Ctrl+D` | 退出 |
| `/mcp` | 管理 MCP 服务器 |
| `/agents` | 管理 Agent |
| `/memory` | 管理记忆 |

### 4.4 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+C` | 取消当前操作 |
| `Ctrl+D` | 退出 |
| `Ctrl+R` | 搜索历史 |
| `Ctrl+O` | 查看详细输出 |
| `Tab` | 自动补全 |

## 五、实际使用场景

### 场景 1：接手新项目

```
你：我刚 clone 这个项目，它是做什么的？
AI：这个项目是... [分析 README、package.json、主要代码]

你：怎么运行它？
AI：我看到 package.json 里有这些脚本...
   建议运行：npm install && npm run dev

你：帮我看看 src/utils 里有什么工具函数
AI：[列出所有文件，总结功能]
```

### 场景 2：改 bug

```
你：用户反馈登录有问题
AI：让我看看相关代码... [搜索 login 相关文件]
   我发现 auth.js 里有个问题...
   需要修改第 45 行的判断逻辑

你：改吧
AI：[显示修改对比] 确认吗？
你：确认
AI：已修改。建议运行测试：npm test
```

### 场景 3：加新功能

```
你：给项目加个用户注册功能
AI：好的，我需要：
   1. 创建注册页面
   2. 添加 API 接口
   3. 更新路由
   计划这样做，可以吗？

你：可以
AI：[一步步执行，每步都显示修改]
```

### 场景 4：代码审查

```
你：帮我 review 这个 PR
AI：好的，让我看看改动... [读取 git diff]
   我发现几个问题：
   1. 第 23 行可能有空指针风险
   2. 缺少错误处理
   3. 建议加单元测试
```

## 六、进阶功能

### 6.1 MCP（接入外部服务）

MCP 让 Claude Code 能连接外部工具：

```
# 连接 GitHub MCP
/mcp add github https://...

# 然后可以直接操作 GitHub
"帮我列出最近的 issues"
"创建一个 PR"
```

### 6.2 Agent（子助手）

复杂任务可以创建专门的子助手：

```
"创建一个专门写测试的 agent 来覆盖这些代码"
"创建 explore agent 来分析项目结构"
```

### 6.3 Plan Mode（计划模式）

大改动时先让 AI 做计划：

```
/plan
"重构整个项目的错误处理"

AI：这是我的计划：
   1. 创建错误基类
   2. 修改 utils/errors.js
   3. 更新所有 API 调用...
   
你：开始执行
AI：[按步骤执行，你可以随时暂停]
```

## 七、最佳实践

### ✅ 要做的

1. **开始新项目时创建 CLAUDE.md** - 写上项目规则和偏好
2. **重要操作前用 /cost 查花费** - 了解 token 消耗
3. **大改动用 Plan Mode** - 避免 AI 一次性改太多
4. **定期用 /compact** - 对话太长时压缩历史
5. **明确告诉 AI 你的约束** - "不要改配置文件"、"保持向后兼容"

### ❌ 不要做

1. **不要在生产环境直接用** - 重要改动先在测试环境验证
2. **不要盲目接受所有修改** - 仔细看对比（diff）
3. **不要把敏感密钥给 AI** - .env 文件保持安全
4. **不要对话太长不压缩** - 会导致变慢和更贵

## 八、常见问题

**Q: Claude Code 会把我代码传走吗？**
A: 只传必要的代码片段给 Anthropic API，不会上传整个项目。

**Q: 支持什么编程语言？**
A: 几乎所有语言都能读，但在 TypeScript/JavaScript、Python、Go、Rust 上表现最好。

**Q: 需要联网吗？**
A: 基本功能需要联网连接 Anthropic API。部分操作（如本地文件编辑）可以离线。

**Q: 和 GitHub Copilot 有什么区别？**
A: Copilot 主要是代码补全，Claude Code 是完整对话式助手，能执行命令、操作文件。

**Q: 免费吗？**
A: 需要 Anthropic API 额度，按 token 计费。可以用 /cost 查看花费。

## 九、总结

Claude Code 把 AI 助手从「聊天窗口」解放出来，直接放到了你的终端里。它能：

- 读你的代码 → 懂上下文
- 改你的文件 → 真干活
- 执行命令 → 深度集成
- 学习你的偏好 → 越用越顺手

**初学者建议**：
1. 先用 Ask 模式，熟悉后再用 Auto
2. 从小改动开始，逐步建立信任
3. 善用 CLAUDE.md 让它记住你的习惯
4. 不懂就问，它是你的编程导师
